# v0.0.2

Released: 2026-05-06

- Added startup Codex auth preflight checks for `serve` and `start`, so missing,
  malformed, expired, or unusable Codex credentials fail before the HTTP server
  binds a port.
- Added visible preflight status output on successful startup to show which
  Codex auth file is being used without exposing token values.
- Improved README usage guidance for `uvx`, daemon commands, default host and
  port, local API-key behavior, licensing, acknowledgements, and PyPI release
  expectations.
- Published the package to PyPI through GitHub Actions Trusted Publishing.
